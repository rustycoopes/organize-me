The first tab renders with the static tab-active/aria-selected="true" markup so the page is already correct in raw server-rendered HTML — before Alpine hydrates — for no-JS clients, screen readers, and plain HTTP fetches. Jinja's autoescaping protects the HTML-attribute boundary; it does not protect the *nested* JS string boundary, so a tab.id containing an apostrophe would break the generated Alpine expression.
